The Big Idea

Every function you've written so far has been alone.
Today, your functions work together.

main() │ ┌─────┴─────┐ ▼ ▼ main_menu() (quit) │ │ │ ▼ ▼ ▼ front back go door door home │ │ ▼ ▼ fight "boring dragon ending"

Each box = one function. Each arrow = a function call.

Two Files, Two Jobs

m5lab1.cpp — The Engine 🔧

You mostly don't touch this.

Contains:

• main() — starts the game

• main_menu() — the menu loop

• global variables — game state

• #include "m5lab1.h"

m5lab1.h — Your Story 📖

This is where you work.

Contains:

• prototypes — function declarations

• definitions — the actual code

• Each function = one story branch

• Add new functions here!

The Pattern — Every New Branch

  1. Name it. What does this branch do?
    Example: choice_fight_dragon
  2. Prototype it. Declare it at the top of the .h file.
    void choice_fight_dragon(); ← semicolon!
  3. Define it. Write the full function below.
    void choice_fight_dragon() { ... } ← no semicolon!
  4. Call it. Add it to the menu or call from another branch.
    case 5: choice_fight_dragon(); break;

What a Branch Looks Like

// In m5lab1.h

// Step 1: Prototype (at the top)
void choice_fight_dragon();

// Step 2: Definition (below)
void choice_fight_dragon()
{
    cout << "The dragon rears back!" << endl;
    cout << "1. Attack with sword" << endl;
    cout << "2. Run away" << endl;

    int choice;
    cin >> choice;

    if (choice == 1) {
        cout << "You strike! -50 dragon HP!" << endl;
        health -= 20;  // global variable
    } else {
        cout << "You escape... barely." << endl;
        choice_run_away();  // call another branch!
    }
}

The Two Bugs You'll Hit

Bug #1: "was not declared"
// You wrote this:
void choice_fight_dragon()
{
    ...
}

// But forgot the prototype!
// Add at the top of .h:
void choice_fight_dragon();
Bug #2: "undefined reference"
// You wrote the prototype:
void choice_fight_dragon();

// But the definition says:
void choice_fight_Dragon()
                   ^^^
// Names must match EXACTLY

Pro tip: Copy-paste function names. Don't retype them.
